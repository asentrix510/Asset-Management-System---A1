This document details the relational database schema of the Asset Management System (AMS), highlighting table attributes, keys, and constraints.

## 2. Table Specifications (Implemented)

### 2.1 The `users` Table
Stores login credentials, contact info, and role permissions.
- **`user_id`** (CHAR(36), Primary Key): Unique string generated by MySQL `UUID()`.
- **`name`** (VARCHAR(100), Not Null): Full name of the employee.
- **`department`** (VARCHAR(100), Null): Corporate department (e.g. IT, HR, Finance).
- **`designation`** (VARCHAR(100), Null): Job title (e.g. Engineer, Lead).
- **`phone`** (VARCHAR(20), Null): User telephone/mobile connection.
- **`email`** (VARCHAR(100), Unique, Not Null): Login email.
- **`password`** (VARCHAR(255), Not Null): Bcrypt password hash.
- **`role`** (ENUM('Admin', 'User'), Not Null): System permission tag.

### 2.2 The `assets` Table
Holds the primary registry of equipment items in stock.
- **`asset_id`** (CHAR(36), Primary Key): Unique asset ID (UUID).
- **`asset_code`** (VARCHAR(50), Unique, Not Null): Tag tracking code (e.g. AST-001).
- **`asset_name`** (VARCHAR(100), Not Null): Description of item.
- **`category`** (VARCHAR(50), Not Null): E.g. Laptops, Phones, Accessories.
- **`brand`** (VARCHAR(50), Null): Manufacturer (e.g., Apple, Dell).
- **`model`** (VARCHAR(50), Null): E.g., MacBook M3, Latitude 5440.
- **`serial_number`** (VARCHAR(100), Unique, Null): Manufacturer serial tag.
- **`purchase_date`** (DATE, Null): Acquisition date.
- **`purchase_cost`** (DECIMAL(10,2), Null): Financial cost of item.
- **`vendor_name`** (VARCHAR(100), Null): Name of supplier (string format).
- **`status`** (ENUM('Available', 'Assigned', 'Maintenance', 'Retired'), Default 'Available'): Active status tracker.
- **`warranty_expiry`** (DATE, Null): Expiration date of coverage.
- **`description`** (TEXT, Null): General notes.
- **`created_at`** (TIMESTAMP, Default CURRENT_TIMESTAMP): System insertion time.
- **`updated_at`** (TIMESTAMP, Default CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP): Last modification time.

### 2.3 The `asset_assignments` Table
Tracks transactions mapping items to user custody.
- **`assignment_id`** (CHAR(36), Primary Key): Unique ID (UUID).
- **`asset_id`** (CHAR(36), Foreign Key -> `assets(asset_id)`, On Delete Cascade): Allocated item.
- **`user_id`** (CHAR(36), Foreign Key -> `users(user_id)`, On Delete Cascade): Custodian.
- **`assigned_date`** (DATE, Not Null): Allocation date.
- **`return_date`** (DATE, Null): Return date.
- **`status`** (ENUM('Assigned', 'Returned'), Default 'Assigned'): Transaction state.
- **`created_at`** (TIMESTAMP, Default CURRENT_TIMESTAMP): Time record logged.
